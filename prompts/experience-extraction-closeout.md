# Experience Extraction Closeout

## 用途

让 agent 在任务完成前，主动判断这次工作是否产生了可复用经验，并把判断结果明确告诉用户，而不是等用户追问。

## Prompt

```text
在任务收尾前，必须追加一次“经验提炼判断”：

1. 回看本次工作是否暴露了重复风险、流程缺口、组件契约规律、review 规则、skill/policy 缺口、harness 入口漂移，或其他未来 agent 仍可能再次犯错的点。
2. 如果有：
   - 明确告诉用户“本次存在可提炼经验”。
   - 说明应该进入 `feedback/`、`prompts/`、`AGENTS.md`、skill，还是机械检查脚本。
   - 若本轮适合直接落地，则在完成实现后立即补记录或补规则，而不是等用户触发。
3. 如果没有：
   - 明确告诉用户“本次没有新增可提炼经验”。
4. 不允许跳过这一步，也不要把“代码已经改好”当作“不需要沉淀经验”的理由。
```

## 效果评价

该 prompt 用于把“任务完成后主动思考是否可沉淀经验，并向用户报告结果”固定为默认收尾动作。它与 `harness-router` 配合：前者负责进入 harness，后者补足收尾时的经验提炼闭环。
