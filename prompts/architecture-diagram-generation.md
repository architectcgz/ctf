# 架构图生成输入包模板

## 用途

当需要把项目文档发给模型生成架构图、Mermaid、PlantUML 或图片时，使用本模板组织输入。目标是让图表只来自当前事实源，避免把 Draft、历史 review 或旧路径画成当前架构。

## 提示词正文

```text
你要根据我提供的 CTF 项目事实源生成架构图。请严格遵守：

1. 只使用“允许事实源”中的内容。
2. “禁止来源”只能作为历史背景，不得画成当前架构。
3. 输出 Mermaid 源码，除非我明确要求 PlantUML / C4 / draw.io。
4. 每个节点和每条边都必须能对应到文档、代码、配置或测试依据。
5. 不确定的内容标为“待确认”，不要补不存在的队列、网关、服务、中间件或缓存。
6. 不输出真实 token、cookie、密码、生产连接串、内网敏感地址；必要时使用 `<token>`、`<database-url>`、`<registry-host>` 占位。

目标图类型：
- 例如：容器图 / 组件图 / 时序图 / 状态图 / 数据流图 / 部署图

允许事实源：
- docs/architecture/...
- docs/contracts/...
- code/backend/...
- code/frontend/...
- scripts/...

禁止来源：
- docs/design/... 中未标记已落地的 Draft
- docs/reviews/... 中的历史 finding
- 已删除路径或 superseded 文档

必须包含：
- 节点：
  - ...
- 边：
  - ...

禁止包含：
- ...

输出要求：
1. 先给 Mermaid 源码。
2. 再列出“节点依据”和“边依据”。
3. 最后列出“不确定项 / 待确认项”。
```

## 使用要求

- 生成前先读 `docs/文档规范.md` 的“架构图生成规范”。
- 输入中的路径必须使用仓库根相对路径。
- 如果图表会进入仓库，文本源码优先保存为 `.diagram.mmd` 或放入 owning 目录的 `diagrams/` 子目录。
