# AWD 学生复盘页结构化 UI 设计

phase12 已经把 AWD 个人攻击事实源接入教师学生复盘接口，但前端页面还停留在“左侧训练时间线 + 右侧证据链”的双日志布局。

这个布局在普通练习阶段还能勉强阅读，一旦 phase12 把 `awd_attack_submit` 和 `awd_attack_submission` 也送进来，页面会出现两个问题：

- 教师需要在一堆按时间混排的事件里自己重建“到底围绕哪道题发生了什么”
- 练习过程和 AWD 对抗过程混在一起，不符合常见 CTF 平台把训练与对抗分开展示的阅读习惯

用户已经明确确认两点：

1. 常见平台不会把练习题和 AWD 题作为同一个前端阅读单元来展示
2. 结构化复盘主视图采用 `A1 案例档案卡`

因此 phase13 的目标不是再补数据，而是把教师学生复盘页改成更像教学复盘的阅读界面。

## 非目标

- 不改后端复盘接口结构
- 不改教师 AWD 比赛复盘页
- 不改个人报告/导出模板
- 不新增新的路由或工作台
- 不在这一轮把 Writeup / Manual Review 完全内嵌进所有案例卡

## 现状问题

### 问题 1：主体区是“日志视角”，不是“案例视角”

当前 `ReviewArchiveEvidencePanel.vue` 用的是：

- 左侧：训练记录
- 右侧：攻防证据链

这更像事件浏览器，不像教师讲一位学生的复盘页。

老师真正想读的是：

- 这位学生在练习里围绕哪些题做了什么
- 在 AWD 里对哪些目标打了哪些题，结果如何
- 哪些题形成了完整的“接入 -> 利用 -> 命中 -> 复盘”闭环

### 问题 2：练习与 AWD 没有被明确分区

当前 phase12 之后，AWD 事件会进入同一组 `timeline / evidence` 数组。

如果前端继续把它们统一铺成一条日志，教师很难快速回答：

- 这是普通训练轨迹，还是比赛对抗轨迹
- 这一条成功命中是练习提交，还是 AWD 攻击命中
- 这一题的复盘材料是对应练习沉淀，还是某次对抗后的整理

### 问题 3：事件粒度过细，缺少卡头摘要

当前事件一上来就是全文 detail，没有先给：

- 本题/本攻击对象的状态
- 最近活动时间
- 是否命中
- 得分
- 有多少证据 / 提交 / 复盘材料

教师要靠扫全量文本才知道重点，阅读成本偏高。

## 参考常见平台后的结论

从 CTFd、picoCTF、TryHackMe、Hack The Box、ICC Attack/Defence Platform 这几类平台的公开资料看，比较稳定的模式是：

- 顶层 summary 可以统一
- 主体阅读区按场景拆开
  - 训练/练习一套
  - 竞赛/对抗一套
- 每个场景内部再按内容单元下钻
  - challenge / room / module / service

因此 phase13 不采用“同一题混合练习与 AWD”的主视图，而采用：

- 顶部统一总览
- 主体分成 `练习复盘` 和 `AWD 复盘`
- 每个分区内部再用案例档案卡承载

## 方案比较

### 方案 A：继续用双日志布局，只给 AWD 事件换颜色

优点：

- 实现成本最低
- 不需要改组件结构

缺点：

- 问题本质没变，还是日志视角
- AWD 数据一多就更难读
- 不能体现“练习 vs 对抗”的阅读边界

不采用。

### 方案 B：统一顶部总览，主体分成练习复盘区与 AWD 复盘区，每个分区都用 A1 案例档案卡

做法：

- 保留现有 Hero / 教学观察 / 训练摘要 / 能力画像
- 用一个新的结构化复盘面板替换原双日志区
- 练习区按 `challenge_id` 聚合
- AWD 区按 `challenge_id + victim_team_name` 聚合
- 每张卡默认折叠，只展示摘要；展开后看阶段和事件

优点：

- 贴近常见平台的场景分区习惯
- 教师一眼能区分训练与对抗
- 便于讲课、复盘、截图和后续导出

缺点：

- 需要新增前端视图模型整理逻辑
- 现有 `timeline/evidence` 组件要重做

推荐采用。

### 方案 C：练习与 AWD 做成两个 tab

优点：

- 切换边界最清楚

缺点：

- 这页已经是深阅读页，再加 tab 会让教师在两块内容间来回切
- 导出/截屏/课堂展示都不如同页分区自然

不采用。

## 目标

本轮完成后：

1. 教师学生复盘页顶部总览保持不变
2. 主体复盘区明确拆成 `练习复盘` 与 `AWD 复盘`
3. 两个分区都使用统一的 `A1 案例档案卡` 视觉语言
4. 练习卡优先表达训练闭环与复盘材料
5. AWD 卡优先表达目标队伍、攻击命中、得分与攻击过程
6. 页面仍保持当前教师端 `teacher-surface` 视觉系统，不变成比赛工作台风格

## 总体设计

### 顶部总览层保持统一

保留：

- `ReviewArchiveHero`
- `ReviewArchiveObservationStrip`
- `训练摘要`
- `能力画像`

原因：

- 这些信息本来就是跨场景汇总
- 教师需要先看到整体判断，再看细部案例

### 主体改成双分区结构化复盘

用新的结构替换现有 `ReviewArchiveEvidencePanel` 的双栏日志布局：

- `练习复盘`
- `AWD 复盘`

两个分区上下排列，而不是 tab 或左右双主栏。

原因：

- 更适合课堂讲解与长页面阅读
- 截图、导出、分享时更完整
- 与现有 `teacher-surface` 页面壳体更兼容

## 数据映射设计

### 练习复盘区

数据来源：

- `timeline`
  - `instance_start`
  - `instance_destroy`
  - `flag_submit`
- `evidence`
  - `instance_access`
  - `instance_proxy_request`
  - `challenge_submission`
- `writeups`
- `manual_reviews`

聚合维度：

- `challenge_id`

一张卡代表：

- 某位学生围绕某道练习题的训练过程与产出沉淀

### AWD 复盘区

数据来源：

- `timeline`
  - `awd_attack_submit`
- `evidence`
  - `awd_attack_submission`

聚合维度：

- `challenge_id + victim_team_name`

一张卡代表：

- 某位学生针对某支目标队伍、围绕某道 AWD 题产生的一组攻击过程

之所以不只按 `challenge_id` 聚合，是因为同一题在 AWD 中可能对多个目标队伍产生不同攻击结果，强行合并会把教学案例压扁。

## A1 案例档案卡设计

### 卡头摘要

所有案例卡都先展示：

- 标题
- 副线
- 结果状态
- 事件数量
- 最近活动时间

练习卡副线优先显示：

- 题目类别 / 训练来源

AWD 卡副线优先显示：

- 目标队伍名

### 阶段摘要

练习卡阶段：

- 接入
- 利用
- 提交
- 复盘

AWD 卡阶段：

- 攻击尝试
- 命中结果
- 得分变化

### 明细展开

默认折叠，只看摘要。

展开后显示：

- 阶段概览条
- 时间顺序事件列表
- 关键元信息
  - 练习：提交结果、分值、材料状态
  - AWD：目标队伍、是否命中、得分

## 视觉与交互边界

### 页面层级

- 一个主 workspace shell
- 两个分区上下排列
- 每个分区有自己的 section head 与辅助说明
- 没有数据时显示显式空态

### 颜色与强调

- 保持教师端 `teacher-surface` 深色表面系统
- 练习卡使用中性/蓝色强调
- AWD 卡在同一主题内使用轻量成功/警示边框与标签
- 不引入比赛大屏式高饱和攻防视觉

### 折叠行为

- 卡片支持展开/折叠
- 默认全部折叠
- 不做复杂联动过滤器
- 移动端保持单列，卡头指标数量压缩

## 文件边界

推荐边界如下：

- `code/frontend/src/components/teacher/review-archive/ReviewArchiveEvidencePanel.vue`
  - 改为结构化复盘主面板
- 新增一个前端本地视图模型整理文件，例如：
  - `code/frontend/src/components/teacher/review-archive/reviewArchiveCases.ts`
  - 负责把现有接口数据整理成 `practice cases / awd cases`
- `code/frontend/src/views/teacher/__tests__/TeacherStudentReviewArchive.test.ts`
  - 增加页面级行为回归
- 新增组件级测试，例如：
  - `code/frontend/src/components/teacher/review-archive/__tests__/ReviewArchiveEvidencePanel.test.ts`

## 测试策略

至少覆盖：

1. 页面会渲染两个分区：
   - `练习复盘`
   - `AWD 复盘`
2. 练习数据不会误入 AWD 分区
3. AWD 数据不会误入练习分区
4. AWD 卡会按 `challenge + victim team` 拆分
5. 卡片默认折叠，展开后能看到阶段与事件明细
6. 教师端 surface 对齐测试仍然通过

## 与毕业设计要求的对应

- “系统自动记录攻击步骤与漏洞利用过程”
  - AWD 分区将把学生对目标队伍的攻击过程收成独立案例卡
- “导出实训报告供教学复盘”
  - 页面会先把复盘阅读结构理顺，为后续导出模板同步奠定基础
- “基于实训数据生成能力画像”
  - 顶部统一总览仍保持练习与 AWD 汇总后的教学视角

## 结论

phase13 不再把新增 AWD 事件简单塞回旧日志布局，而是把教师学生复盘页正式改成“统一总览 + 分开的练习复盘区与 AWD 复盘区 + A1 案例档案卡”的结构化页面。这更符合常见平台对训练与对抗的分区方式，也更适合课堂讲解。
