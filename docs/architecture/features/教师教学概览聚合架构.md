# 教师教学概览聚合架构

> 状态：Current
> 事实源：`code/backend/internal/module/teaching_readmodel/**`、`code/frontend/src/features/teacher-dashboard/**`、`code/frontend/src/components/teacher/dashboard/TeacherDashboardPage.vue`
> 替代：无

## 定位

本文只定义教师侧 `/academy/overview` 的总览聚合边界，以及它与 `/academy/classes/:className` 班级详情工作区的职责分工。

本文不负责：

- 教师权限模型从单班级扩展到多班级的数据库设计
- 班级详情页的完整趋势、复盘、洞察、介入面板规则
- 教学建议规则内核本身；建议生成规则继续由 `教学复盘建议生成架构.md` owning

## 当前设计

### 总览与详情的 owner

- `/academy/overview`
  - 负责：展示教师当前可访问教学范围内的摘要信号，包括范围指标、聚合趋势、重点班级、重点学生和薄弱维度分布
  - 不负责：直接承载某个班级的完整趋势图、完整复盘结论列表、完整推荐题块或学生详情分析链路

- `/academy/classes/:className`
  - 负责：展示单个班级的完整趋势、复盘、学生洞察和介入建议
  - 不负责：代表教师全局教学范围的摘要结论

### 后端 query owner

- `teaching_readmodel.QueryService.GetOverview`
  - 负责：按“当前请求者可访问的班级范围”输出 overview 级读模型
  - 不负责：返回班级详情页所需的完整 review item、推荐题明细或学生个人画像

- `teaching_readmodel.QueryService.GetClassSummary / GetClassTrend / GetClassReview`
  - 负责：单班级详情读模型
  - 不负责：为 overview 直接提供聚合展示 contract

### 前端 owner

- `useTeacherOverviewPage`
  - 负责：加载 overview 专属接口、处理失败状态和进入班级管理跳转
  - 不负责：继续拼接 `getClassSummary/getClassTrend/getClassReview` 形成伪总览

- `useTeacherOverviewWorkspace`
  - 负责：把 overview DTO 转成摘要卡片、趋势信号、班级聚焦和介入列表所需的视图模型
  - 不负责：承载 class-level 详情文案拼装

- `TeacherDashboardPage.vue`
  - 负责：渲染 overview 级工作区
  - 不负责：直接挂载 `TeacherClassTrendPanel`、`TeacherClassReviewPanel`、`TeacherInterventionPanel`

## 边界

### 总览 scope

教学概览的基本 scope 是“当前请求者可访问的班级集合”。

- 当前教师权限模型仍然来自 `users.class_name` 单字段，因此普通教师 today 只会看到 1 个班级 scope。
- overview contract 不能把这个偶然现状固化成“必须传入单个 class_name”。
- 后续若教师权限扩展到多班级，overview contract 保持不变，只扩大 `accessible classes` 的求值来源。

### 详情 scope

班级详情继续显式依赖 `className`。

- 完整复盘建议、推荐题和单班级趋势只属于详情 scope。
- overview 只能消费摘要级衍生结果，例如重点班级排序、近 7 天聚合趋势、优先跟进学生。

## 接口与契约影响

新增 overview 专属接口：

- `GET /api/v1/teacher/overview`

返回 contract 按 summary scope 组织，至少包含：

1. `summary`
   - `class_count`
   - `student_count`
   - `active_student_count`
   - `active_rate`
   - `average_solved`
   - `recent_event_count`

2. `trend.points`
   - 近 7 天聚合活跃学生、训练事件、成功解题

3. `focus_classes`
   - 班级名
   - 学生数
   - 活跃率
   - 近 7 天事件数
   - 重点薄弱维度
   - 风险学生数

4. `focus_students`
   - 学生标识
   - 班级名
   - solved / score / recent_event_count / weak_dimension

5. `weak_dimensions`
   - 薄弱维度
   - 命中学生数

这个接口不返回：

- `TeacherClassReviewItem[]`
- 班级详情推荐题明细
- 学生能力画像细节

## 关键链路

1. 用户进入 `/academy/overview`
2. `TeacherDashboard.vue` 仅组合 `useTeacherOverviewPage`
3. `useTeacherOverviewPage` 调用 `getTeacherOverview`
4. `useTeacherOverviewWorkspace` 把 overview DTO 映射成 summary tabs 所需视图模型
5. `TeacherDashboardPage.vue` 渲染 overview 级 tab 内容
6. 若用户需要完整复盘或学生详情，再从班级管理进入 `/academy/classes/:className`

## 兼容与迁移

- 当前数据库与权限模型仍是单班教师绑定 `class_name`；本次不在同一任务里改 RBAC 或表结构。
- 即便当前多数教师只看到 1 个班级，overview 页面也不再直接复用 class-level 详情组件。
- 旧的 `/academy/overview` 前端实现是“默认班级工作台”；本次迁移后，它改为真正的 summary workspace。
- 班级详情页保持原有 class-level 读模型与 panel owner，不在本专题里回收。
