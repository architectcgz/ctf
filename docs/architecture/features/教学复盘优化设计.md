## 1. 背景与问题

教师学生分析页当前已经不是单纯的“若干列表”。代码里已经落成了一套把画像、推荐、题解、人工审核、证据链、攻击会话和归档快照组织到同一工作台内的个人教学复盘入口。

这里需要明确的，不是再次重复证据事件语义或会话切分细节，而是这几个 shared 专题在教师工作台中如何分工：

- 实时教学复盘同时消费平铺证据流和攻击会话，但这两条读模型各自已有 owning doc。
- 完整归档页与实时分析页共享同类教学事实，但职责不同，不能混成一页内的多重状态机。
- AWD 个人事件需要进入教学复盘工作台，但不能直接改写练习型 `solved / score / rank` 指标。

## 2. 架构结论

- `TeacherStudentAnalysis.vue` 是教师个人教学复盘的实时工作台入口，承载画像、推荐任务、发布题解、人工审核、证据链和训练记录。
- 证据事件语义与脱敏规则由 `攻击证据链与教学复盘架构.md` owning；攻击会话的分组键、切分窗口和结果判定由 `攻击会话读模型与复盘工作台架构.md` owning。
- 当前工作台自身只负责把这两条读模型和页面筛选条件组织到同一套前端状态中，不在本专题内再次定义事件类型或会话切分规则。
- `TeacherStudentReviewArchive` 继续保留为稳定归档页；通用报告任务生命周期由 `赛事导出与复盘归档架构.md` owning，实时分析页不复用归档页的本地状态机。
- 教学建议和推荐理由规则由 `教学复盘建议生成架构.md` owning；本专题只说明这些结果如何挂到教师工作台，而不是重新定义建议内核。
- 个人 / 班级成绩口径当前仍按练习 solved / score / rank 统计；AWD 个人事件只进入证据链、攻击会话、归档快照和建议生成链路，不直接改写这些练习型排名指标。

## 3. 模块边界与职责

### 3.1 模块清单

- `teaching_query.QueryService`
  - 负责：向教师工作台暴露证据链与攻击会话查询，并做教师侧权限校验
  - 不负责：维护建议规则或报告生命周期

- `useTeacherReviewWorkspace`
  - 负责：加载 `evidence`、`attackSessions`、维护共享筛选状态和 challenge 选项
  - 不负责：重新实现证据语义或攻击会话切分

- `useTeacherStudentAnalysisPage`
  - 负责：加载画像、推荐、时间线等基础工作台数据，并协调各标签页入口
  - 不负责：直接组织 evidence / session 的筛选联动逻辑

- `StudentAnalysisPage`
  - 负责：把画像、推荐、题解、人工审核、证据链、训练记录组织成多标签工作台
  - 不负责：自行拼装后端查询

- `TeacherReviewArchiveWorkspace`
  - 负责：展示 `review-archive` 返回的稳定快照
  - 不负责：复用实时工作台的筛选和会话重算

### 3.2 事实源与所有权

- 工作台编排事实源：`TeacherStudentAnalysis.vue` + `StudentAnalysisPage`
- 实时证据事实源：`GetStudentEvidence`
- 会话化事实源：`GetStudentAttackSessions`
- 归档事实源：`ReportService.GetStudentReviewArchive`

## 4. 关键模型与不变量

### 4.1 核心实体

- 工作台标签集合
  - `学员画像`
  - `推荐任务`
  - `发布的题解`
  - `人工审核`
  - `证据链`
  - `训练记录`

- 共享筛选条件
  - `challenge_id`
  - `contest_id`
  - `round_id`
  - `mode`
  - `result`
  - `with_events`

- 工作台消费的主要数据片段
  - `TeacherEvidenceResp.events[]`
  - `TeacherAttackSessionResp.sessions[]`
  - `ReviewArchiveData`

### 4.2 不变量

- 本文不重复定义证据事件类型、`meta` 结构和脱敏规则；这些事实由 `攻击证据链与教学复盘架构.md` owning。
- 本文不重复定义会话分组键、`1h` 切分窗口和 `success / failed / in_progress / unknown` 判定规则；这些事实由 `攻击会话读模型与复盘工作台架构.md` owning。
- 共享筛选条件必须同时作用于 evidence 与 sessions，不能让两块内容各自维护不同的 challenge / contest / round 口径。
- 前端 challenge 筛选选项来自 evidence 与 sessions 的合并结果，不单独维护静态字典。
- `TeacherStudentReviewArchive` 是稳定快照页，不复用实时工作台的本地筛选状态。
- AWD 个人事件会进入教学复盘读模型，但不会把个人 / 班级报告里的 `solved / score / rank` 直接改写成比赛口径。

## 5. 关键链路

### 5.1 实时教学复盘链路

1. 教师进入 `TeacherStudentAnalysis.vue`。
2. `useTeacherStudentAnalysisPage` 加载画像、推荐、时间线等基础数据。
3. `useTeacherReviewWorkspace` 先调用 `getStudentEvidence`，再调用 `getStudentAttackSessions`。
4. `StudentAnalysisPage` 在“证据链”和“训练记录”标签中分别展示平铺事件与攻击会话。

### 5.2 共享筛选链路

1. 教师在工作台切换 challenge、contest、round、mode 或 result。
2. `useTeacherReviewWorkspace` 统一更新筛选状态。
3. 前端按同一组筛选条件重载 `evidence` 与 `attack-sessions`，保持两块内容的阅读上下文一致。
4. 具体事件过滤、会话聚合和结果判定仍由各自 owning doc 对应的后端查询层负责。

### 5.3 归档链路

1. 教师可从分析页跳转到完整复盘页，或直接导出 review archive。
2. 归档页 `TeacherStudentReviewArchive` 读取 `ReportService.GetStudentReviewArchive` 的稳定快照。
3. 实时页与归档页共享同类教学事实，但归档页不再重新计算实时筛选状态。

## 6. 接口与契约

### 6.1 教师复盘接口

- `GET /teacher/students/:id/evidence`
- `GET /teacher/students/:id/attack-sessions`
- `GET /teacher/students/:id/review-archive`

其中共享筛选字段包括：

- `mode`
- `challenge_id`
- `contest_id`
- `round_id`
- `result`
- `with_events`
- `limit`
- `offset`

### 6.2 前端工作台契约

`StudentAnalysisPage` 当前稳定承载以下标签：

- `学员画像`
- `推荐任务`
- `发布的题解`
- `人工审核`
- `证据链`
- `训练记录`

## 7. 兼容与迁移

- 证据流仍然保留平铺结构，便于审计与归档；会话只是读取层派生视图。
- AWD 赛事复盘页和学生分析页仍分别服务“赛事态势”和“个人教学复盘”，不合并为同一条路由。
- 若后续要调整 session 切分窗口、事件语义或建议规则，应先更新对应 owning doc，而不是在本文里追加第二套定义。
