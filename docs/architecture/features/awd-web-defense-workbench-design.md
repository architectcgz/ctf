# AWD Web 防守工作台设计

## 背景

学生侧 AWD 战场已经具备本队服务状态、攻击目标、stolen flag 提交、防守 SSH、服务重启和最近事件。但当前“防守”仍主要依赖 SSH 或浏览器文件工作台。浏览器文件工作台如果直接挂到学生页面，会把 `app.py`、目录结构和容器路径暴露给参赛者，不适合作为默认入口。

本设计把学生防守入口从“文件浏览器”调整为“Web 防守工作台”：学生先看服务风险、攻击痕迹和受控修复区，再执行验证和重启。SSH 保留为高级入口，不作为完成防守的唯一方式。

## Gemini CLI 调用记录

本轮按要求调用 Gemini CLI 参与页面设计：

- 第一次：`gemini -p ...`，默认模型连续返回 `500 backendError` 和 `429 MODEL_CAPACITY_EXHAUSTED`。
- 第二次：`gemini -m gemini-2.5-flash -p ...`，返回 `ECONNRESET`，未能取得模型设计结果。

因此本设计以本地代码、既有 AWD 架构文档和当前安全约束为依据继续推进。Gemini 调用失败不改变页面设计原则：默认学生入口不能暴露完整源码、完整目录树、真实容器路径或任意命令执行。

## 目标

- 学生从 `/contests/:id?panel=challenges` 的 AWD 攻防战场进入防守工作台。
- 防守区提供服务状态、风险排序、攻击痕迹、修复动作、验证动作和高级连接入口。
- 默认不展示完整 `app.py`、完整目录树、真实容器路径、flag、token、secret。
- 后续代码片段能力必须由后端策略生成，不能由前端传任意 path 读取。
- SSH 作为高级入口保留，Web 工作台作为默认学生防守路径。

## 非目标

- 不做完整 Web IDE、在线 VS Code 或浏览器 shell。
- 不开放任意文件读写、任意目录浏览、任意命令执行。
- 不把管理员流量分析、容器运维字段或 readiness 动作搬到学生侧。
- 不改变 AWD checker、计分、轮次调度和重启语义。
- 不保证学生永远无法通过 SSH 查看源码；若赛制开启 SSH，SSH 本身就是强能力入口。本设计只约束平台 Web UI/API 的默认暴露面。

## 入口

学生仍从竞赛详情页进入：

```text
/contests/:id?panel=challenges
```

AWD 赛事下该页签显示为“攻防战场”。页面左侧或上方的“我的防守”区域成为 Web 防守入口：

- 服务卡点击后选中服务。
- 选中服务下方展示该服务的防守工作台。
- 工作台内分为风险、修复、验证、连接四个区块。

不新增 `/contests/:id/awd/defense` 独立路由，避免学生在比赛详情页和防守页之间来回跳转。

## 信息架构

### 本轮态势

顶部 HUD 保留当前回合、本队、服务数、榜首和同步状态。它回答“当前战场是否还在跑、数据是否新鲜”。

### 我的防守

防守区优先级高于攻击区，服务按风险排序：

1. `compromised`
2. `down`
3. 收到攻击
4. 重启、创建、恢复中
5. 正常

服务卡展示：

- 题目名
- 服务编号
- checker 状态
- 实例状态
- 风险标签
- 打开服务
- Web 防守
- 重启
- SSH

### 服务防守工作台

选中服务后展示四个连续区块。

#### 风险片段

风险片段不是文件列表，而是平台允许学生看到的防守线索：

- 异常状态：失陷、离线、待同步、重启中
- 攻击线索：最近受到攻击次数、最近事件结果
- 可疑入口：题目作者或策略配置的逻辑位置，例如“登录接口”“文件上传处理”“订单查询接口”
- 受控代码片段：只展示策略允许范围，默认最多几十行

`app.py` 不作为文件名暴露给学生。后端可以把它抽象成：

```text
登录处理
上传处理
用户查询
模板渲染
```

片段响应只返回逻辑位置、语言、起止行的相对标识、脱敏后的代码内容和是否可编辑，不返回真实容器路径。

#### 补丁区

补丁区只允许编辑后端策略指定的范围：

- 不允许保存完整文件。
- 不允许创建新文件。
- 不允许跳转到未授权路径。
- 保存前后端生成备份和审计记录。
- 保存接口绑定 `contest_id / team_id / service_id / fragment_id`。

一期不实现真正保存补丁，只做 UI 入口和禁用态；二期再接后端策略 API。

#### 验证区

验证区承接学生修复后的动作：

- 打开本队服务
- 刷新战场
- 重启服务
- 查看最近 checker / 攻击事件是否恢复

验证不等于执行任意命令。第一阶段只复用已有打开、刷新和重启能力。

#### 连接区

连接区保留 SSH：

- SSH 不是默认防守路径。
- 生成 SSH 后展示 `ssh user@host -p port` 和 OpenSSH 配置。
- SSH 标记为高级入口，适合本地编辑器或线下赛制。
- 若平台配置关闭 SSH，Web 防守工作台仍可使用状态、风险和验证能力。

## 安全边界

后端必须保证：

- 所有防守接口必须校验登录用户属于当前 contest 的当前 team。
- 只能访问本队当前 service，不能访问他队 service。
- 片段由服务端策略决定，前端不能传任意 path。
- 不返回真实容器路径、完整目录树、完整源码、secret、flag、token。
- 片段内容经过脱敏。
- 保存补丁前生成备份。
- 每次读取片段、保存补丁、重启服务都写审计记录。
- 片段大小、补丁大小、请求频率和输出大小都有上限。

## API 方向

一期不新增危险读写 API，只复用现有 workspace、SSH、open service、restart service。

二期新增受控片段 API，而不是启用通用 files/directories：

```text
GET /api/v1/contests/:id/awd/services/:sid/defense/fragments
GET /api/v1/contests/:id/awd/services/:sid/defense/fragments/:fragment_id
PUT /api/v1/contests/:id/awd/services/:sid/defense/fragments/:fragment_id/patch
POST /api/v1/contests/:id/awd/services/:sid/defense/fragments/:fragment_id/validate
```

片段 DTO 示例：

```json
{
  "fragment_id": "login-handler",
  "title": "登录处理",
  "language": "python",
  "risk_level": "warning",
  "status": "readonly",
  "editable": false,
  "content": "def login():\\n    ...",
  "redacted": true,
  "updated_at": "2026-05-04T13:00:00Z"
}
```

## 前端一期范围

一期先交付可用 Web 防守工作台外壳：

- 学生页不再挂完整文件工作台。
- “防守监控”调整为“我的防守”。
- 服务卡增加“Web 防守”动作和清晰的 `SSH` 高级入口。
- 选中服务后展示防守工作台：
  - 风险状态
  - 攻击线索
  - 片段区占位：仅展示“等待策略片段”，不展示文件名或路径
  - 验证动作：打开服务、刷新、重启
  - 高级连接：复用 SSH 面板
- 攻击区与情报区保持在同一战场页面，不新增学生路由。

## 二期范围

二期再做受控代码片段和补丁：

- 后端新增片段策略配置。
- 题目包或 service 配置声明允许展示的片段。
- 后端读取容器文件时按策略裁剪、脱敏和审计。
- 前端渲染片段列表和只读代码块。
- 受控 patch 保存、备份和验证。

## 验收标准

- 学生从攻防战场可直接看到“我的防守”和服务防守工作台。
- 学生页没有完整文件工作台、目录树或完整源码读取入口。
- 页面不会显示 `app.py`、真实容器路径或任意文件路径。
- SSH 仍可生成，但只是服务卡的高级连接能力之一。
- 重启、打开服务、刷新仍可从防守工作台完成。
- 前端类型检查和相关组件测试通过。
