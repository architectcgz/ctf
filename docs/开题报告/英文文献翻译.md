## 一 A Review of Cyber Ranges and Test Beds Current and Future Trends

原文题目 A Review of Cyber Ranges and Test Beds Current and Future Trends
来源 Sensors 2020

中文题目 网络靶场与测试平台综述 现状与未来趋势

摘要翻译
网络态势感知已经被证明能够帮助组织形成对威胁与脆弱性的全面理解 组织暴露程度受到网络卫生水平和既有流程的影响 更准确的安全评估能够识别最脆弱的环境并指导更严格的管理 随着网络攻击自动化快速扩散 信息技术与运行技术之间的界限不断缩小 迫切需要回顾面对新型复杂攻击时的稳健性 并研究趋势 技术 以及缓解对策 更深入的特征刻画也有助于预测未来脆弱性 从而指导更合适的部署技术 因此需要更新既有实践与训练范围 以支撑用户与运维人员的决策训练的基础是网络靶场与测试平台 这类平台工具可以帮助学习者更深入理解攻击的演化过程以及如何部署更有效的对策以阻止入侵 本文对已有文献中的网络靶场与测试平台进行评估 并按类型 技术 威胁场景 应用与可达训练范围进行分段分析 为了丰富分析并形成收束 本文提出一个分类体系 以帮助理解网络靶场与测试平台的未来 该分类体系阐述了多个维度 并指出不同应用领域之间的差异正在减弱

引言要点翻译节选
作者引用 NIST 对网络靶场的定义 认为网络靶场是对组织本地网络 系统 工具 与应用的交互式模拟表示 测试平台被定义为可配置可扩展并包含工业过程仿真的网络靶场 网络靶场主要面向信息与通信技术人员训练 测试平台主要面向运行技术网络的人员训练 两者随着物联网与工业物联网发展而更加交织 通过在可识别的仿真环境中构建攻击场景 能够提升操作者对入侵识别与处置的能力 并通过实时交互促进对行为后果的理解

与本课题相关的启示整理
1 平台能力需要从 训练对象 场景类型 技术实现 交互方式 与评估范围 等维度刻画 便于做需求边界与路线规划
2 教学型靶场强调可重复可控与可度量 平台应内置数据采集与评估出口
3 趋势上容器化与云化会降低交付成本 但安全边界与隔离机制仍是核心挑战

## 二 Experimental Analysis of Security Attacks for Docker Container Communications

原文题目 Experimental Analysis of Security Attacks for Docker Container Communications
来源 Electronics 2023

中文题目 Docker 容器通信安全攻击的实验分析

摘要翻译
Docker 作为开源容器平台被广泛用于应用打包与运行 由于具备可移植性与轻量化等优势 尤其受到提供云服务的企业与 IT 开发者关注 Docker 通过内部网络配置为多个容器提供通信 使得可以将容器在逻辑上互联来更容易地组合各种服务 但近来出现了利用 Docker 容器网络脆弱性的网络攻击 例如分布式拒绝服务与加密货币挖矿攻击 本文通过实验展示攻击者如何在容器环境中执行 ARP 欺骗 DDoS 与权限提升等攻击 并从网络流量 CPU 消耗 与恶意反向 shell 执行等角度对结果进行分析 此外 通过从 Docker 容器环境的网络视角审视这些攻击 作者为检测与防范容器之间可能发生的横向移动攻击奠定基础

论文贡献点翻译
1 聚焦 Docker 容器网络 给出在最新 Docker 环境中仍可执行多种攻击的实际验证
2 给出 ARP 欺骗 DDoS 权限提升等攻击的详细实验步骤 并基于网络流量 CPU 消耗 恶意反向 shell 执行对结果进行分析

与本课题相关的启示整理
1 不能仅从应用层考虑平台安全 需要将容器通信与网络层攻击纳入威胁模型
2 默认采用每实例独立网络 并尽量减少共享二层广播域 可以降低 ARP 欺骗与横向移动的影响面
3 需要对异常流量与资源消耗建立监控点 在竞赛高并发下尤其重要

## 三 The Docker Security Playground A hands on approach to the study of network security

原文题目 The Docker Security Playground A hands on approach to the study of network security
来源 IPTComm 2017

中文题目 Docker 安全游乐场 面向网络安全学习的动手式方法

摘要翻译
本文提出并实现 Docker Security Playground 该架构利用基于微服务的方法构建复杂网络基础设施 以支持网络安全研究与学习该系统从一开始就被定位为一种动手学习网络安全的工具 作者已经实现并在公共仓库中提供了一系列安全实验室 并讨论学生如何有效使用这些实验 同时也介绍了面向程序员的应用编程接口 以支持开发新的实验

引言要点翻译节选
作者指出网络安全研究与实践往往涉及复杂网络基础设施 漏洞评估一般遵循 信息收集 扫描与枚举 利用 后利用 等阶段 为支持这些操作已经出现大量工具 Docker 通过轻量容器为本地部署安全服务提供了灵活方式 结合镜像仓库可以实现服务的快速分发与更新 该工作强调在合法合规前提下构建隔离的虚拟实验环境 以降低学习与实验成本

与本课题相关的启示整理
1 容器化与微服务方式适合构建可复用的安全实验内容 便于教师与学生共享与迭代
2 通过公共仓库管理实验内容的模式适合在校园内形成题库与实验库的持续建设机制
3 平台需要提供对实验与题目的可扩展接口 以便后续引入更多场景与拓扑

## 四 Cyber Operations RangE CORE Containerized Gaming Platform for Cyberoperations Competitions

原文题目 Cyber Operations RangE CORE Containerized Gaming Platform for Cyberoperations Competitions
来源 2017

中文题目 网络作战靶场 CORE 面向网络作战竞赛的容器化游戏平台

摘要翻译
网络安全游戏通过 CTF 竞赛 战争游戏 与取证挑战而广受欢迎 这类竞赛能够提供真实的动手经验 对课堂学习 专业技能提升 与场景测试具有很大潜力 要充分发挥这些价值 竞赛支撑平台需要解决以下问题 第一 复杂的虚拟化基础设施需要大量资源与管理 第二 缺少对竞赛环境中实时网络作战的支持 第三 需要实现独立的实时计分系统以跟踪进度并确定结果 为解决这些问题 作者开发了 CORE 平台 这是一个容器化的网络安全竞技平台 本文介绍 CORE 平台的设计 实现 以及其在实时网络作战竞赛中的应用

相关工作与设计要点翻译节选
作者指出传统攻击防守类竞赛需要复杂的虚拟机基础设施 资源消耗随参赛者数量增长 CORE 通过容器化降低资源与运维复杂度 并强调实时对抗与更细粒度的实时计分 使参赛者能够更快看到攻防动作带来的效果

与本课题相关的启示整理
1 平台需要同时考虑训练型 Jeopardy 与对抗型 AWD 的组织差异 并在一期交付中预留扩展能力
2 实时计分与状态检测是竞赛体验的关键 需要在架构上支持高频的状态更新与推送
3 容器化可以显著降低竞赛环境的资源成本 但仍需结合隔离策略与审计能力保证平台安全
