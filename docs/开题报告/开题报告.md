# 一、选题依据

## （一）研究背景与意义

近年来，网络空间安全威胁呈现“高频化、自动化、链条化”的趋势。攻击者借助自动化工具与漏洞供应链，能够在较短时间内完成扫描、入侵、横向移动与持续控制，且攻击目标从传统信息系统扩展到云平台、物联网与关键基础设施。高校在网络安全人才培养方面承担重要职责，但在实际教学过程中常出现以下矛盾：一是理论学习与实战能力之间存在断层，学生对真实攻击链路、漏洞利用与防护闭环缺乏体验；二是训练环境建设成本高、维护难度大，传统虚拟机靶场在资源占用、交付效率、隔离管理等方面难以长期支撑校园规模化训练。

CTF（Capture The Flag）竞赛与网络靶场训练能够将知识点转化为可操作、可验证的任务，覆盖漏洞分析、利用与修补、权限控制、日志审计、取证与溯源等能力点，并以积分与排名的方式形成即时反馈与激励机制。面向教育与训练的网络靶场与测试平台已经被大量研究与实践证明具有价值：Ukwandu 等对网络靶场与测试平台的定义、分类与发展趋势进行了系统梳理，为构建训练平台能力维度提供了参考［1］；Perrone 等提出基于容器的教学实验平台 DSP，说明容器化能够降低实验交付成本并提升可维护性［3］；而竞赛平台方面，CORE 的容器化竞赛组织与实时计分思想对高校开展竞赛具有借鉴意义［4］。同时，容器网络自身也存在可被利用的攻击面，Lee 等对 Docker 容器通信的攻击实验表明，ARP 欺骗、DDoS 与权限提升等攻击在容器环境中仍可能发生［2］，这提示我们必须将“靶机环境的风险外溢”视为平台建设的首要约束。

基于以上背景，本课题拟在校园网内部署一套“教学训练 + 校赛竞赛 + 能力评估”的一体化平台：在满足教学可用性的同时，强调容器隔离、最小权限与可审计性，保证靶机漏洞不影响宿主机与平台服务；在满足竞赛实时性与公平性的同时，提供动态 Flag 与反作弊基础机制；在满足教学管理需求的同时，提供能力画像与评估报告导出，为教学改进提供数据支撑。

## （二）国内外研究现状

1 国外研究现状。  
在网络靶场与测试平台研究方面，Ukwandu 等对 CR 与 TB 的概念、类型、应用领域、训练对象与未来趋势进行了综述，并指出不同应用领域间边界趋于模糊，平台形态呈现融合与演进［1］。在教学与实验平台方面，DSP 强调以容器与微服务方式构建可复用实验室，并通过仓库共享实验内容与接口扩展能力［3］。在竞赛平台方面，CORE 从容器化降低资源与运维复杂度，并强调实时作战与实时计分体验［4］。在容器安全研究方面，Lee 等通过实验从容器网络角度分析攻击步骤与影响，提示需要从网络与资源消耗侧建立检测与防护思路［2］。

2 国内研究现状。  
国内研究多围绕教学实训平台、虚拟仿真靶场与行业靶场展开。张晨从教学实训角度给出基于虚拟化容器技术的实验平台设计［7］；梁建辉等基于 Docker 进行靶场工程化实现，强调容器化对交付效率的提升［8］；周云等探讨综合靶场建设与管理体系［6］；刘安等面向 Web 应用靶场建设提出方案与建议［5］；同时，面向特定行业（如铁路）的靶场建设也形成了可借鉴的工程经验［10］。总体而言，现有研究为靶场建设提供了基础，但面向校园的“一体化训练竞赛评估闭环”与“容器安全边界细化”仍有提升空间。

3 现有不足与本课题定位。  
多数方案要么偏向企业级与行业级场景，部署与运维复杂；要么偏向单一训练平台，缺少竞赛组织与评估体系。针对校园约束，本课题选择模块化单体架构，以降低复杂度并提升交付效率；在功能上优先交付日常训练与 Jeopardy 校赛核心链路，并预留 AWD 扩展；在安全上以容器隔离与最小权限为硬约束，确保靶机风险不外溢；在教学上以能力画像与报告导出形成数据闭环。

# 二、研究内容、目标与拟解决的关键问题

## （一）研究目标

总体目标：设计并实现一套校园级网络攻防竞赛平台，在校园网内部署运行，采用容器化提供隔离的漏洞靶机环境，支撑日常训练与校赛组织，并提供可量化的技能评估能力。  
分目标包括：  
1 完成用户认证与 RBAC 权限控制，满足管理员、教师、学生的差异化权限需求；  
2 完成靶场管理能力，支持题目、镜像、难度与标签、附件与 Writeup 的管理；  
3 完成攻防演练闭环，支持实例启动、访问、续期与回收，以及 Flag 提交判定与积分记录；  
4 完成竞赛管理能力（一期 Jeopardy），支持队伍管理、实时排行榜与冻结榜；  
5 完成技能评估能力，形成按分类与难度维度的能力画像，并支持报告导出；  
6 完成容器管理基础设施，提供容器与网络的统一封装，并强制资源限制与隔离策略。

## （二）研究内容

1 平台业务模块研究。  
围绕靶场管理、攻防演练、竞赛管理与技能评估四个主模块进行需求分析、数据建模与实现方案设计，明确各模块边界、数据流与接口协议，并形成可验收的用例集合。

2 容器生命周期与隔离机制研究。  
围绕容器创建、启动、停止、销毁、过期回收与异常清理等环节，设计实例状态机与幂等控制；围绕网络隔离、端口映射、资源限制、权限最小化等策略构建安全边界，确保靶机风险不影响平台。

3 动态 Flag 与反作弊基础机制研究。  
研究动态 Flag 的生成、注入与校验方法，确保“每用户每题唯一”与“可复算可追溯”；结合提交频率限制与审计记录，降低暴力提交与答案共享风险。

4 实时计分与排行榜机制研究。  
研究得分计算、排名维护、冻结榜展示与 WebSocket 推送机制，满足竞赛实时性与一致性要求。

5 能力画像与评估报告研究。  
研究基于解题数据的能力画像生成方法，建立按题型分类与难度维度的评价模型；研究班级与时间段的统计口径与导出结构，形成可用的教学评估输出。

## （三）拟解决的关键问题

1 如何在保证学员可访问靶机的前提下，构建“靶机与平台服务”之间的安全边界，并抵御容器网络侧的横向移动风险［2］。  
2 如何实现动态 Flag 的生成、注入与校验，使其满足唯一性、可追溯性与可复算性，同时兼顾实现复杂度与性能。  
3 如何在并发场景下保障容器启动与回收的稳定性，避免资源泄露与异常状态堆积。  
4 如何实现实时计分、排行榜推送与冻结榜策略，确保竞赛公平性与可用性。  
5 如何将训练与竞赛数据转化为可解释、可导出的能力画像与评估报告，形成教学闭环。

# 三、研究方案、技术路线与可行性分析

## （一）研究方案概述

平台采用模块化单体架构（Modular Monolith）与分层设计：Handler 层负责路由、参数校验与响应封装；Service 层负责业务编排、事务边界与权限判断；Repository 层负责数据访问与缓存读写；Docker Engine 层封装容器与网络管理能力。基础设施层采用 PostgreSQL 作为主存储，Redis 用于会话、缓存、限流与排行榜；前端采用 Vue 3 单页应用，通过 WebSocket 获取实时得分与排名变动。

## （二）技术路线

本课题按“文献调研—需求建模—原型验证—工程实现—实验评估”的路线推进：  
1 文献调研：以网络靶场能力维度与发展趋势为依据，结合国内外容器化平台经验与容器通信攻击研究，形成平台目标与关键风险清单［1-4］。  
2 需求建模：梳理教学训练与竞赛组织的核心用例，输出可验收需求与非功能指标（启动时间、并发、延迟与隔离要求）。  
3 原型验证：优先验证容器创建、独立网络隔离、端口映射、动态 Flag 注入与提交判定链路。  
4 工程实现：在既定模块边界下实现核心模块与共享基础设施，并完成前后端联调。  
5 实验评估：通过并发压测、长稳运行与故障注入验证性能与可靠性，并结合安全隔离测试验证边界有效性。

## （三）关键技术方案

1 容器隔离与资源限制。  
每个用户实例默认创建独立 Docker Network，禁止跨实例互访；容器对外仅通过宿主机动态端口映射提供访问入口，并限定端口范围；强制禁止特权容器并尽量采用最小权限运行；设置 CPU、内存、进程数等资源限制，降低异常行为对平台的影响。

2 动态 Flag。  
以题目模板与系统盐值为基础，结合用户标识与实例标识生成动态 Flag；实例启动时将 Flag 注入容器指定位置；平台侧保留可验证的生成参数或可重算摘要；提交判定使用统一校验逻辑，并加入提交频率限制与审计记录。

3 实时计分与排行榜。  
服务端完成得分与排名维护，Redis 用于实时排行榜；通过 WebSocket 推送排名与得分变动；冻结榜在展示层与真实榜之间隔离，兼顾公平性与可解释性。

4 能力画像与评估报告。  
基于用户解题数据（题型分类、难度分布、通过率、耗时等）生成能力画像数据；教师侧按班级与时间段导出评估报告，为教学改进提供依据。

## （四）可行性分析

技术可行性：平台技术栈成熟，Go 与 Vue 生态完善；Docker 容器化在教学与竞赛中已有实践基础［3-4］。  
条件可行性：可在校园服务器部署，使用 Docker Engine 运行靶机容器，使用 PostgreSQL 与 Redis 提供数据与缓存支撑。  
实施可行性：模块化单体降低开发与运维复杂度，一期聚焦训练与 Jeopardy 核心链路，二期再扩展 AWD，符合毕业设计周期。  
安全可行性：以独立网络隔离、最小权限与资源限制为硬约束，并以审计与限流降低风险外溢可能，符合容器网络安全研究给出的风险提示［2］。

# 六、预期成果与创新点

## （一）预期成果

1 一套可运行的校园级 CTF 网络攻防竞赛平台（支持日常训练与 Jeopardy 校赛）。  
2 系统架构设计文档、API 设计文档、数据库设计文档、部署与运维说明。  
3 英文文献翻译稿件与面向开题的文献综述材料。  
4 毕业设计论文与答辩展示材料。  

## （二）创新点

1 训练—竞赛—评估一体化：在同一平台内形成闭环，减少系统拼装成本，提高可运营性。  
2 默认强隔离与最小权限：以每实例独立网络与资源限制为硬约束，并结合容器通信攻击研究完善边界设计［2］。  
3 数据驱动能力画像：将解题数据转化为可解释的能力画像与评估报告，支撑教学改进。  
4 动态 Flag 与反作弊基础：在保证可复算与可追溯的前提下实现动态 Flag，并配套限流与审计策略。
