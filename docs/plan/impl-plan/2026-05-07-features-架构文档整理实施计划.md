# features 架构文档整理实施计划

## Objective

把 `docs/architecture/features/` 从“专题设计与过程混合目录”收口为“最终专题架构事实源”，并把中间收口稿里的最终结论并回正式架构文档。

## Non-goals

- 这轮不批量重命名仍作为最终事实源的架构专题文件名。
- 这轮不把所有混合型专题一次性拆成多份文档；先完成入口规则和首批可直接收口的专题。
- 这轮不修改运行时代码、接口契约或页面实现。

## Source Inputs

- `AGENTS.md`
- `docs/architecture/README.md`
- `docs/architecture/features/专题架构索引.md`
- `works/harness-good-practices.md`
- 已采纳的专题架构文档模板

## Chosen Direction

按两类专题分批迁移：

1. 已接近架构事实源的专题，直接重写为统一架构样式。
2. 仍混合页面设计、方案比较、实施顺序的专题，先在目录入口中明确它们后续需要拆分到 `docs/architecture/frontend/pages/`、`docs/design/` 或 `docs/plan/impl-plan/`，暂不在本轮整批强行压成一份“假架构文档”。
