## 新发现问题

### 🔴 高优先级

#### [H8] 定时任务启动失败未阻止服务启动
- **文件**：`internal/app/http_server.go:32-34`
- **问题描述**：`cleaner.Start()` 失败时返回错误，但如果 cron 表达式配置错误，整个服务将无法启动
- **影响范围/风险**：配置错误导致服务不可用，影响可用性
- **修正建议**：
```go
if err := cleaner.Start(cfg.Container.CleanupInterval); err != nil {
    log.Error("启动清理任务失败，将使用默认间隔", zap.Error(err))
    // 降级：使用默认间隔
    if err := cleaner.Start("*/5 * * * *"); err != nil {
        return nil, fmt.Errorf("启动清理任务失败: %w", err)
    }
}
```

### 🟡 中优先级

#### [M8] 资源泄漏：HTTPServer 关闭时未等待清理任务完成
- **文件**：`internal/app/http_server.go:54-55`
- **问题描述**：`cleaner.Stop()` 立即停止 cron，但可能有正在执行的清理任务
- **影响范围/风险**：清理任务被中断，可能导致状态不一致
- **修正建议**：
```go
func (s *HTTPServer) Shutdown(ctx context.Context) error {
    s.logger.Info("停止清理任务")
    s.cleaner.Stop() // cron.Stop() 会等待正在执行的任务完成
    return s.server.Shutdown(ctx)
}
```
注：经查 `robfig/cron/v3` 的 `Stop()` 方法已经会等待任务完成，当前实现可接受，但建议添加注释说明。

#### [M9] 依赖管理：新增依赖未在文档中说明
- **文件**：`go.mod:10`, `go.sum:153-154`
- **问题描述**：新增 `github.com/robfig/cron/v3` 依赖，但 commit message 中虽有提示需要 `go mod download`，但未在架构文档中记录
