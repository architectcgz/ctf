- 历史 review 文件中的行号、提交范围、测试数量和“未修复”状态只代表该文件审查日期对应的代码状态。

- 已合并到主线的修复应优先写回主索引，不在旧 review 原文中反复改写历史结论。
- 低优先级体验建议如果已经被后续产品方向覆盖，应在主索引中标记为“过期 / 已由后续实现覆盖”，而不是继续重复排期。
