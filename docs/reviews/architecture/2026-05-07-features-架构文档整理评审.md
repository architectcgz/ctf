这轮收口已经把 `features/` 目录中最后一批混合型专题压回到当前代码事实，而且处理方式是正确的：

- `AWD防守工作区与边界设计` 现在明确写成当前 SSH-only 防守入口、`defense_workspace` 题包约束和 `/workspace` 登录边界，而不是继续保留浏览器文件工作台方案。
- `AWD学员实战工作台设计` 现在明确锚在 `ContestDetail -> ContestAWDWorkspacePanel -> GET /contests/:id/awd/workspace` 这条已上线链路上，不再保留独立学生 AWD 页面设想。
- `社区题解与推荐题解设计` 现在明确锚在 `recommended / community / my writeup`、solved gate、`draft / published` 和教师运营动作上，不再把旧评阅流当当前主模型。
- `竞赛题目编排工作台设计` 现在明确写清当前只有 `basics / pool / awd-config / preflight` 四段，运行态仍归 `AWDOperationsPanel`，避免重新误读成单页全流程运营台。
- `CTF前端试探型彩蛋设计` 现在明确写清它只是 `sessionStorage` 本地态和五个已落地探测点，不再保留 ARG 式扩展方案。
- `校园级CTF-AWD模式完整设计` 现在已经从“目标蓝图”改成当前实现总览，清楚区分资源层、赛事配置层、运行编排层、裁判计分层、学生战场层和教师复盘层。

另外，这轮顺手把“单题题包设计不属于 `features/`”这条规则也写回了入口文档和 `works/harness-good-practices.md`。这一步是必要的，因为它能阻止后续再把某一道具体题目的题面/解法稿误塞回架构专题目录。

到这里，`features/专题架构索引.md` 的“需要拆出页面设计或中间设计的混合型专题”列表已经清空。对这个目录本身来说，本轮结构债已经基本收口。
