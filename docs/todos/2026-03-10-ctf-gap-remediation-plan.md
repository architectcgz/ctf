# CTF 平台缺口修复清单

更新日期：2026-03-12

## 四模块主线收敛结论（2026-03-12）

基于原始需求基线 `/docs/requirements/ctf-platform-requirements-baseline.md`，当前后续开发应只按以下四个模块判断是否偏题：

1. 靶场管理
2. 攻防演练
3. 竞赛管理
4. 技能评估

结合当前代码实现，主线收敛结论如下：

### 1. 靶场管理

- 已形成主线闭环：
  - 管理员挑战 CRUD、发布、Flag 配置、附件、Hint、题解已打通。
  - 难度分级、分类、镜像选择、拓扑编排、环境模板、多网络分段与 ACL 已落地。
  - 运行时多节点实例启动与网络回收已接入真实容器链路。
- 当前未发现阻断原始需求的硬缺口。

### 2. 攻防演练

- 已形成主线闭环：
  - 学员可选择靶场、启动实例、获取攻击目标并提交 Flag。
  - 平台已记录读题、Hint 解锁、实例启动/续期、HTTP 访问利用轨迹、提交结果等训练过程。
  - 练习链路已接入实时得分反馈与能力画像增量更新。
- 当前未发现阻断原始需求的硬缺口。

### 3. 竞赛管理

- 已形成主线闭环：
  - Jeopardy/AWD 竞赛基础链路、报名审核、公告、个人进度、队伍模式、动态计分已落地。
  - AWD 已具备轮次、服务检查、攻击日志、防守成功统计、实时排行榜与管理端实时监控面板。
  - 当前“人工补录攻击日志不计入正式榜单”的口径已修正，避免复盘数据污染正式成绩。
- 当前未发现阻断原始需求的硬缺口。

### 4. 技能评估

- 已形成主线闭环：
  - 能力画像、薄弱维度识别、针对性靶场推荐已实现。
  - 教师侧班级概览、趋势、复盘结论、优先介入学生、建议训练题已实现。
  - 报告支持页面直接预览，下载导出改为可选动作，符合“先看再下”的要求。
- 当前未发现阻断原始需求的硬缺口。

### 结论

- 原始四模块当前已基本闭环。
- 下面仍未勾选的事项，默认不再视为“四模块主线必须项”，除非后续再次确认要扩展验收口径。
