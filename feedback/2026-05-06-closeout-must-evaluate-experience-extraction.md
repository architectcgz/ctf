# 任务收尾必须判断是否可提炼经验

## 问题描述

agent 在完成实现、验证和提交后，容易把“代码改完”当成收尾终点，遗漏一次明确的经验提炼判断，结果需要用户反复追问“这次是否应该写进 harness / skill”。

## 原因分析

现有 harness 更强调任务开始时的 intake、执行中的验证和改动后的 consistency check，但没有把“收尾前主动思考是否产生可复用经验，并向用户报告”写成默认闭环动作。

缺少这一步会导致：

- 经验沉淀依赖用户提醒，而不是 agent 主动完成。
- 有些可复用规则已经在实现里收口，却没有进入 `feedback/`、`prompts/` 或 skill。
- 用户需要额外承担 harness 维护判断。

## 解决方案

把“经验提炼判断”加入 harness prompt 作为默认收尾步骤：

- 任务完成前必须判断本次是否产生了可复用经验。
- 如果有，明确告诉用户并说明应沉淀到哪里。
- 如果没有，也要明确报告“本次没有新增可提炼经验”。

本次已把这条要求写入 `prompts/experience-extraction-closeout.md`，并补进 `prompts/harness-router.md` 的 HARNESS 收尾步骤。

## 收获

Harness 不只需要入口分流，还需要出口闭环。经验提炼判断应成为默认收尾动作，而不是用户额外承担的提醒职责。
