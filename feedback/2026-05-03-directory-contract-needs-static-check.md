# 目录规范变更需联动静态校验

## Migrated From

## Status

not-impl

## 问题描述

统一题目包 Dockerfile 位置时，先改了题目目录、示例包和文档，但遗漏了上传/导入静态校验。

## 原因分析

目录规范如果只改说明和示例，不改 parser/import service、前端格式说明、分发包和测试，旧结构可能继续通过上传校验，造成文档和运行行为漂移。

## 解决方案

以后凡是调整题目包结构、文件位置、命名规范或目录约定，必须同步检查：

- 后端 package parser/import service。
- 前端格式说明。
- 示例题目包。
- dist 分发包。
- 相关测试。

## 收获

目录规范属于契约，不只是文档。契约变更应配套静态校验和测试。
