# 页面设计：Writeup 管理与查看

> 继承：design-system/MASTER.md | 任务：T22
> 技术栈：Element Plus (主) + Tailwind CSS (辅)
> 管理员端位置：管理后台 > 靶场管理 > Writeup Tab
> 学员端位置：靶场详情页 > Writeup 区域

---

## 技术栈

**Element Plus：** `<ElTable>`, `<ElDrawer>`, `<ElInput>`, `<ElSelect>`, `<ElRadioGroup>`, `<ElDatePicker>`, `<ElButton>`
**Tailwind CSS：** 布局、间距

## 3. 学员端 — Writeup 查看

在靶场详情页（challenges.md）底部，已解出靶场新增 Writeup 区域：

```
┌──────────────────────────────────────────────────────────┐
│  (靶场详情页现有内容: 题目信息、实例控制、Flag 提交、提示)  │
├──────────────────────────────────────────────────────────┤
│  Writeup                                                 │
│  ┌────────────────────────────────────────────────────┐  │
│  │ SQL注入基础解题思路                    作者: admin  │  │
│  │ ────────────────────────────────────────────────── │  │
│  │ ## 解题思路                                        │  │
│  │                                                    │  │
│  │ 1. 首先观察登录表单的参数传递方式...                  │  │
│  │ 2. 尝试经典的 SQL 注入 payload...                   │  │
│  │ 3. 获取数据库中的 flag 表...                        │  │
│  │                                                    │  │
│  │ (Markdown 渲染，代码块使用 font-mono + 语法高亮)     │  │
│  └────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

### 可见性规则

| 条件 | 显示 |
|------|------|
| 未解出 + 未公开 | 不显示 Writeup 区域 |
| 未解出 + 已公开 | 显示，但折叠，需点击展开 + 剧透警告 |
| 已解出 + 已公开 | 直接显示 |
| 竞赛进行中 | 无论状态均不显示 |

### 剧透警告（未解出但已公开）

```
bg-warning/5 border border-warning/20 rounded-lg p-4 cursor-pointer

[EyeOff 20px text-warning]
"此 Writeup 包含解题过程，查看前建议先自行尝试"
[点击展开]

展开后: 正常渲染 Markdown 内容
```
