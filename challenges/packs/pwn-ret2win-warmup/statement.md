# Pwn 入门：ret2win 热身

这是一个最基础的栈溢出热身题。服务启动后会等待你输入名字，但读取长度明显超过了栈上缓冲区大小。

程序里存在一个不会被正常流程调用的 `win()` 函数。请通过覆盖返回地址跳转到 `win()`，读取实例中的动态 Flag。

## 目标

1. 下载或导出题目二进制并确认保护情况。
2. 找到 `win()` 函数地址。
3. 构造 payload 覆盖返回地址。
4. 连接实例服务并获取动态 Flag。

## 访问方式

- 容器服务端口：`8080/tcp`
- 本地测试示例：

```bash
docker run --rm -p 18080:8080 -e FLAG='flag{local_ret2win_test}' <image>
nc 127.0.0.1 18080
```

## 提示

- 这是 64 位 ELF。
- `win()` 会从环境变量 `FLAG` 读取动态 Flag。
- 常见偏移可以从 `64 字节缓冲区 + saved rbp` 的方向验证。
