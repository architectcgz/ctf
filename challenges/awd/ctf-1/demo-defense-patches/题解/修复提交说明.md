# AWD 三题修复提交说明

这份说明对应 `demo-defense-patches` 目录下的三份修复版代码，目的是让你在演示时可以直接说明：

- 原文件改哪里
- 用哪份修复代码替换
- 每题的关键修复点是什么

## 1. 供应链工单系统

原文件位置：

- `challenges/awd/ctf-1/awd-supply-ticket/docker/app/app.py`

修复版位置：

- `challenges/awd/ctf-1/demo-defense-patches/awd-supply-ticket/app.py`

建议提交方式：

- 直接用修复版替换原题的 `app.py`

这题主要改了四件事：

1. `SECRET_KEY` 不再固定弱值，改为环境变量优先，缺省时随机生成。
2. 默认管理员不再硬编码 `admin123`，只有设置了 `ADMIN_PASSWORD` 才创建管理员。
3. `/notify/<id>` 不再对工单标题做二次 `render_template_string`。
4. 工单标题和内容增加长度限制与 HTML 转义。

演示时可以重点指这几个函数：

- `init_db()`
- `sanitize_text()`
- `new_ticket()`
- `notify()`

## 2. 校园网盘

原文件位置：

- `challenges/awd/ctf-1/awd-campus-drive/docker/app/app.py`

修复版位置：

- `challenges/awd/ctf-1/demo-defense-patches/awd-campus-drive/app.py`

建议提交方式：

- 直接用修复版替换原题的 `app.py`

这题主要改了四件事：

1. 上传文件不再保留原始落盘名，统一改成随机文件名。
2. 增加 `BLOCKED_SUFFIXES`，拒绝双扩展脚本类文件。
3. 新增 `META_DIR` 保存“展示名 -> 实际文件”的映射。
4. 预览接口从 `GET /preview?path=...` 改成 `GET /preview/<file_id>`，不再接受原始路径。

演示时可以重点指这几个位置：

- `is_allowed_upload()`
- `upload()`
- `resolve_storage_name()`
- `preview()`

## 3. IoT 设备管理平台

原文件位置：

- `challenges/awd/ctf-1/awd-iot-hub/docker/app/app.py`

修复版位置：

- `challenges/awd/ctf-1/demo-defense-patches/awd-iot-hub/app.py`

建议提交方式：

- 直接用修复版替换原题的 `app.py`

这题主要改了四件事：

1. 不再使用全局默认设备密钥，改为 `devices` 表按设备存储密钥。
2. `telemetry` 与 `topic_read` 都改成结构化解析 topic，不再做裸前缀判断。
3. `/api/command` 需要 `X-Admin-Token`。
4. `/firmware` 只允许从 `firmware_files` 白名单读取，不直接拼路径。

演示时可以重点指这几个函数：

- `setup()`
- `require_device()`
- `split_topic()`
- `telemetry()`
- `command()`
- `firmware()`

## 4. checker 兼容性

这三份修复版都保留了：

- `GET /health`
- `PUT /api/flag`
- `GET /api/flag`

也就是保留了你当前 AWD 演示赛依赖的 checker 路径，不会因为修补把比赛直接修挂。

如果你现场只想一句话讲明白，可以直接这么说：

- `awd-supply-ticket`：替换 `docker/app/app.py`，重点修模板注入、默认口令、会话密钥和输入转义。
- `awd-campus-drive`：替换 `docker/app/app.py`，重点修危险上传和预览路径问题。
- `awd-iot-hub`：替换 `docker/app/app.py`，重点修默认密钥、Topic ACL 和固件路径读取。
