# AWD 防守修复示例

这组文件用于课堂或答辩演示三道 AWD 题目的防守修复思路。

目录说明：

- `awd-supply-ticket/app.py`：供应链工单系统修复版
- `awd-campus-drive/app.py`：校园网盘修复版
- `awd-iot-hub/app.py`：IoT 设备管理平台修复版

使用约定：

- 这些文件是单独的演示补丁，不直接覆盖原题源码。
- 保留了 `/health` 和 `/api/flag`，方便继续通过现有 checker。
- 修复重点是“最小闭环防守”，不是做完整生产级重构。

每题对应的防守重点：

- `awd-supply-ticket`
  - 去掉默认弱口令
  - 会话密钥必须外部注入
  - 通知页不再二次模板渲染
  - 输入增加长度限制和转义

- `awd-campus-drive`
  - 上传文件统一重命名
  - 禁止双扩展和可疑后缀
  - 预览改为文件 ID，不再接受原始路径
  - 下载和预览都只走映射后的受控文件

- `awd-iot-hub`
  - 设备密钥改为按设备记录
  - Topic 查询改为结构化校验
  - 控制指令需要管理口令
  - 固件下载只允许白名单对象
