Passkey Sync Gateway 对外暴露了一套文本 TCP 协议，用来在多端之间同步恢复事件和审计快照。

开发同学为了排障，保留了一条“导出调试快照”的命令。请像真实 AWD 一样分析这套协议逻辑，利用漏洞获取当前实例中的动态 Flag。

## 目标

1. 分析协议命令集合与业务判断逻辑。
2. 利用调试导出命令读取当前实例中的动态 Flag。
3. 在平台提交本轮 Flag。

## 访问入口

- `tcp://<host>:8080`

## 补充说明

- checker 使用 `PING / SET_FLAG / GET_FLAG` 完成动态 Flag 管理。
- 防守时只修改业务工作区代码，不要破坏 runtime 中的 checker 管理逻辑。
