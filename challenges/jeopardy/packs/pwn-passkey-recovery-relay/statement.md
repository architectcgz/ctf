一套内部 Passkey 恢复中继服务会读取“恢复短语”，再转发到下游系统。

实际程序对输入长度完全不设防，而且还留了一个不会被正常调用的恢复函数。请分析附件里的 64 位 ELF，构造 payload 覆盖返回地址并读取实例中的动态 Flag。

## 目标

1. 分析附件 `relay.bin` 的保护情况和关键函数地址。
2. 找到恢复函数入口。
3. 构造 payload 覆盖返回地址。
4. 连接实例服务并获取动态 Flag。

## 访问方式

- 容器服务端口：`8080/tcp`
- 本地测试示例：

```bash
docker run --rm -p 18085:8080 -e FLAG='flag{local_passkey_relay_test}' <image>
nc 127.0.0.1 18085
```

## 提示

- 这是 64 位 ELF。
- 恢复函数会从环境变量 `FLAG` 读取动态 Flag。
- 常见偏移可以从 `64 字节缓冲区 + saved rbp` 的方向验证。
