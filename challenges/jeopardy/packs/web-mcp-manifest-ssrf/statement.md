团队最近做了一个很简陋的 MCP Connector Manifest 预览器，方便在发布前检查第三方工具的 `manifest.json`。

这个预览器会主动抓取你输入的 URL，再把返回内容原样展示出来。开发同学只加了非常粗糙的本地地址黑名单，但内部还留了一个仅允许本机访问的调试接口。

请分析这个预览器的抓取逻辑，访问到内部调试接口，并拿到实例中的动态 Flag。

## 目标

1. 访问首页，确认 manifest 预览入口。
2. 利用预览器访问仅限本机访问的内部接口。
3. 在平台提交拿到的动态 Flag。

## 访问入口

- `/` 首页
- `/preview?url=<manifest-url>` 预览接口

## 补充说明

- 内部接口不会在首页直接暴露。
- 预览器的黑名单不是按 DNS 解析结果做的。
